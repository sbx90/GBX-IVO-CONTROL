Business Flows

🛠️ Kit Lifecycle
1
Create Kit
Staff enters serial + type → DB trigger auto-creates 9 components + 13 mainboard sections for MAIN_BOARD
2
Kit status: OK
Default state. Board map shows all sections, POWER_2 always pulses red as warning
3
Ticket created on kit
If kit.status === OK → auto-updated to TICKET (client-side in useCreateTicket)
4
Kit status: DEAD
Manually set when kit is unrecoverable. Shown with red badge
🎫 Ticket Flow
1
OPEN
Created. Optional: linked to client, component, board section
2
IN_PROGRESS
Staff picks it up. Known issues panel suggests solutions
3
RESOLVED
resolved_at timestamp auto-set
4
CLOSED
Final state. Can attach images to comments throughout
⚡ POWER_2 Warning System
The #1 installation error is plugging into POWER_2 (right connector) instead of POWER_MAIN (left). This triggers:

• Board map: POWER_2 always has red pulsing border (.danger-pulse CSS animation)
• Ticket form: selecting POWER or WRONG_CONNECTOR category shows amber alert: "Verify power cable is in LEFT connector (POWER_MAIN)"
• Known issues: "Wrong Power Connector Installation" seeded as HIGH frequency issue
🏭 Production Flow
1
Create Order (QUEUED)
order# + quantity + target date → 10 steps auto-created by DB trigger
2
ACTIVE
Steps: Component Sourcing → PCB Assembly → CM4 Installation → Camera Install → Power Wiring → USB Config → Antenna Wiring → Door Lock → Firmware Flash → Final Inspection
3
COMPLETE
Step 10 marked DONE → order automatically set to COMPLETE
4
CANCELLED
Manually set. Shown in kanban with separate column
🔐 Auth Flow
1
proxy.ts (middleware)
Every request checked for valid session. Unauthenticated → redirect /login. Excludes: _next/static, _next/image, favicon, /login
2
AuthErrorHandler (providers.tsx)
On mount: checks for stale/invalid refresh token → auto sign-out → redirect /login
3
onAuthStateChange
SIGNED_OUT event → redirect /login
🚀 Deploy Flow
1
DeployButton (sidebar)
Click → POST /api/deploy
2
API route
GET latest Vercel deployment → POST redeploy with same deploymentId
3
Result
Button shows: deploying → deployed! → link to gbx-ivo-control.vercel.app
